# X/Twitter Channel Rules

## Tone
Punchy, opinionated, occasionally dry humor. Confident but not arrogant. Every tweet should make a developer stop scrolling.

## Constraints
- **Single tweets only**: Maximum 280 characters. Count carefully. Do NOT generate threads.
- **Hashtags**: Maximum 2 per tweet. Prefer none unless genuinely useful for discovery.
- **Emojis**: Sparingly. One at most. Never emoji-heavy.

## Hook Patterns That Work
- Lead with capability: "AI agents can now pay for APIs autonomously. No human in the loop. x402 handles it."
- Counterintuitive claim: "API keys are the new fax machines."
- Relatable pain point: "Every agent framework has a 'payment' gap. Nobody knows how to fill it."
- "We shipped X" + what it means (e.g., "Agents can now set on-chain spending limits through the dashboard. Squads SPN, no config required.")
- Protocol adoption signal (x402 has 5400+ GitHub stars, AP2 has 60+ partners, UCP has 20+ partners)
- Cost comparison is a secondary angle, not the primary hook. Lead with what's now possible, not what's cheaper.

## Content Angles for Current Release (v0.14.0)
- Squads SPN spending limits: agent operators can set maxPerTransaction and maxPerDay
- On-chain Smart Accounts created directly in the PincerPay dashboard (Phantom/Solflare)
- Three-layer spending controls: SDK, facilitator, on-chain
- The competitive angle: non-custodial vs custodial (Skyfire), stablecoin vs card rails (ACP/Stripe)
- Protocol stack: UCP discovery -> AP2 trust -> x402 settlement -> chain abstraction

## Anti-Patterns (NEVER do these)
- No "GM" or "gm" posts
- No engagement bait: "What do you think?", "Like if you agree", "Drop a fire emoji"
- No hashtag spam: #Web3 #AI #Crypto #Payments #DeFi
- No emoji-laden hype
- No vague promises: "Big things coming soon"
- No corporate announcements: "We're thrilled to announce..."
- No price/token speculation (there is no token)

## Voice Examples

Good tweet:
"Three lines of code. Your agent now pays for APIs on its own. No card, no human, no approval flow. Just x402."

Bad tweet:
"Exciting news! PincerPay is revolutionizing the future of AI payments! Check out our amazing new feature! #AI #Web3 #Payments"

Good tweet:
"Agents on PincerPay can now set spending limits on-chain. Three layers: SDK cap, facilitator enforcement, Squads SPN. No single point of failure."

Bad tweet:
"We're so excited to share what we've been building at PincerPay! It's been an incredible journey!"
